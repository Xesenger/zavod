Алгоритм названия смены (ZAVOD v1)

Активная смена:
- fixed title: "Текущая смена"
- подзаголовок: живой ориентир по текущему смыслу

Подзаголовок активной смены собирается так:
1. если есть validated intent / active task:
   -> берём его краткую формулировку
2. иначе если есть candidate intent высокой зрелости:
   -> берём его краткий смысл
3. иначе:
   -> "Ожидает следующего шага"

Примеры:
- Текущая смена
  работаем над экраном проекта
- Текущая смена
  дорабатываем lifecycle результата
- Текущая смена
  ожидает следующего шага

Закрытая смена:
- title формируется только после closure
- источник истины:
  closure summary
  accepted results
  decisions
  final task outcomes
- shift становится read-only архивом

Алгоритм title для закрытой смены:
1. собрать "что реально было главным результатом смены"
2. убрать шум:
   - уточнения
   - промежуточные ревизии
   - разговоры
3. выбрать один главный outcome
4. сжать до короткого названия 3–7 слов
5. если outcome неочевиден:
   -> брать самый сильный подтверждённый structural change
6. если смена была mostly alignment/discussion:
   -> назвать по итоговому решению, а не по болтовне

Формула:
title = итог смены
subtitle/summary = что именно изменили

Правила:
- не использовать первое сообщение
- не использовать сырые фразы пользователя
- не использовать "обсуждали", если был реальный итог
- не делать слишком общие названия типа "Работа над проектом"
- название должно читаться как запись в архиве

Шаблоны title:
- Экран проекта и смены
- Closure review для смен
- Result lifecycle и abandon path
- Intent validation flow
- Архив закрытых смен

Если итог технически слабый или смешанный:
приоритет такой
1. decision
2. accepted result
3. closure summary
4. active task final outcome

UI модель:
- Активная:
  "Текущая смена"
  + живой подзаголовок
- Закрытая:
  финальное название
  + короткая summary строка
