# ZAVOD v1 — Current Shift Trace

## Purpose

`Current Shift Trace v1`
- описывает наблюдательный слой текущей смены
- фиксирует процесс работы внутри смены
- служит источником данных для closure и snapshot
- не является частью ProjectState

---

## Definition

`Current Shift Trace` — это последовательность событий и наблюдений,
связанных с активной сменой.

Trace отражает:
- что происходило в смене
- какие действия выполнялись
- какие решения обсуждались или принимались

---

## Nature

- trace является observer-only слоем
- trace не изменяет ProjectState напрямую
- trace не считается истиной проекта
- trace может содержать шум и промежуточные состояния

---

## Content (v1)

Trace может включать:

- рабочие действия (implementation steps)
- изменения в задачах (task progress)
- зафиксированные результаты (accepted results)
- обсуждения и формулировки намерений (intent evolution)
- принятые решения (если явно зафиксированы)
- сигналы ошибок / отклонений (optional)

---

## Structure

В v1 структура trace не нормализована строго:

- допускается событийная форма (event stream)
- допускается частично структурированный формат
- допускается наличие неиспользуемых или шумовых элементов

Главное требование:
- trace должен позволять восстановить ход смены на уровне смысла

---

## Relation to Closure

- closure summary формируется на основе trace + shift state
- пользовательское уточнение дополняет trace перед финальной сборкой
- trace используется для определения:
  - уровня изменений (локальные / структурные / фундаментальные)
  - необходимости обновления документов

---

## Relation to Snapshot

- snapshot не равен trace
- snapshot является сжатым результатом смены
- trace может использоваться как источник для snapshot,
  но не переносится в него полностью

---

## Canons

- trace не является источником истины проекта
- trace не записывается в ProjectState
- trace не должен напрямую изменять документы проекта
- только зафиксированные решения могут влиять на базовые документы
- отсутствие явного решения в trace означает отсутствие фундаментальных изменений

---

## Boundaries

- trace не описывает lifecycle смены
- trace не заменяет task model
- trace не выполняет анализ или интерпретацию сам по себе
- trace не гарантирует полноту или точность без closure слоя

---

## Exclusions

- строгая схема данных (strict schema)
- гарантированная чистота данных
- автоматическая фильтрация шума
- использование trace как пользовательского интерфейса
