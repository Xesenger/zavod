# ZAVOD v1 — Decision Model

## Purpose

`Decision Model v1`
- описывает механизм фиксации решений внутри смены
- определяет, какие изменения считаются осознанными
- служит единственным источником изменений для базовых документов проекта

---

## Definition

`Decision` — это явно зафиксированное решение,
принятое в рамках смены и влияющее на проект.

Decision отражает:
- что было решено
- в каком контексте
- какое изменение это вносит

---

## Nature

- decision является смысловой фиксацией изменения
- decision не равен обсуждению
- decision не выводится автоматически без достаточных оснований
- decision может отсутствовать в смене

---

## Structure (v1)

Decision содержит:

- `Description` — что было решено
- `Context` — при каких условиях / почему
- `Impact` — на что влияет решение (roadmap / canon / stack / etc.)

---

## Origin

Decision может появиться:

- из задачи (task → decision)
- из обсуждения (discussion → decision)
- из явного утверждения пользователя

---

## Relation to Trace

- decision должен быть отражён в trace
- отсутствие явной фиксации в trace означает отсутствие decision
- обсуждение без фиксации не считается решением

---

## Relation to Closure

- closure слой использует уже зафиксированные decisions
- только decisions могут инициировать изменения базовых документов
- decisions учитываются при формировании closure payload

---

## Relation to Documents

Decision может привести к обновлению:

- roadmap
- canon
- stack / архитектуры
- других стабильных документов проекта

---

## Canons

- только decision может изменять базовые документы проекта
- отсутствие decision означает отсутствие фундаментальных изменений
- decision должен быть явно сформулирован
- decision не должен выводиться из косвенных сигналов
- system должна действовать консервативно при интерпретации decision

---

## Boundaries

- decision не описывает выполнение задач
- decision не заменяет task model
- decision не гарантирует корректность реализации
- decision не является частью ProjectState напрямую

---

## Exclusions

- автоматическое извлечение решений без подтверждения
- неявные или предположительные решения
- множественные уровни согласования (approval chains)
- голосования / коллективные decision механики
