ZAVOD UI RECOVERY CONTEXT
Date: 2026-04-11

1. What actually broke
- The app stopped launching reliably after a failed root UI pass.
- The visible blast radius was:
  - App.xaml
  - MainWindow.xaml
  - MainWindow.xaml.cs
- Observed failures included:
  - Windows App SDK startup crash before App.OnLaunched
  - wrong Visual Studio launch profile
  - blank/white main window after startup

2. What was fixed already
- Unpackaged launch recovery is in place.
- Project startup profile is back to normal Project launch.
- The app now builds and launches again.
- Pass 1 visual host extraction is done:
  - theme dictionaries split out of App.xaml
  - mode switch extracted
  - Chats / Projects / Project List / Project Home / Project Work Cycle visual hosts extracted
- Pass 2 is done:
  - WorkCycleActionController is the active owner for the discussion/preflight action path
- Pass 3A is done:
  - existing Projects / Conversation / Markdown owners are relocated under UI/*
- Pass 3B / Pass 4 Lite and Pass 4B are done:
  - obvious host-owned glue has moved into extracted views
  - MainWindow no longer owns every visual tree directly
- MainWindow is still the temporary shell/composition host and still owns mixed shell glue on purpose.

3. What recovery understanding was wrong
- The older recovery interpretation treated the inner 3-surface shell as if it were the app-level model.
- That is not the right product split.
- The correct top-level split is:
  - Chats
  - Projects

4. Correct product model
- Chats:
  - free chats
  - not ZAVOD project lifecycle
- Projects:
  - the actual entry into ZAVOD
  - contains:
    - Project List
    - Project Home
    - Project Work Cycle

5. Correct Projects model
- Project List:
  - import
  - create new project
  - choose project
- Project Home:
  - simple overview layer
  - HTML summary button
  - scanner/import material list
  - simple statuses
- Project Work Cycle:
  - inner ZAVOD core
  - Chat / Execution / Result model

6. Important distinction
- Top-level Chats mode is not the same as Chat surface inside Project Work Cycle.
- Project Work Cycle chat is bounded project workflow UI.
- Free Chats is separate product behavior.

7. Prompt / streaming scope
- Three-Part Prompt Assembly & Anchor System belongs only to Projects / Project Work Cycle.
- Execution Streaming & Chat Projection Layer belongs only to Projects / Project Work Cycle.
- Free Chats mode must stay outside that bounded project execution model.

8. Repo mutation rule
- Real repo changes must appear only after:
  - Accept
  - in Result
- Navigation and shell switching must not mutate repo truth.

9. Cold start rule
- Cold start is valid.
- Empty state must not confuse or break ZAVOD.
- Cold start also covers future New Project entry scenarios where the user has weak initial understanding.

10. Status of older recovery notes
- Older recovery notes may still contain useful low-level hints.
- But they may describe the wrong top-level model.
- Use them with distrust and re-verify against code before relying on them.
